Introduction to Python Decorators

// under python decorators

Decorators are supported by the Python language, and can wrap functions and classes, adding extra capabilites or management functionality (e.g. checking if the user is logged in, before executing a function). In this post, the basic concepts and syntax of Python decorators will be introduced.

Introduction

A decorator is callable object (e.g. function or class method), which allows the user to augment or add management functionality to functions and classes. Decorators are an extension to the Python language syntax, they were first introduced in PEP 318, and implemented in Python 2.4. Python decorators are related to the decorator design pattern, however they also support other use cases, e.g. static methods in classes.

Decorator functionality can also be manually implemented (e.g. helper functions), however there are some advantages to their usage. Decorator syntax is explicit, which makes your code more idiomatic and easier to maintain, since it is easier to spot a decorator than a call to a helper function inside the code, especially if your function or class is complex. They are applied once, when the function or class is defined, thus eliminating the need to repeat the decorator code (which may change), whenever the decorated function or class is called or instantiated. Therefore, decorators facilitate code maintenance and encapsulation. There are also some drawbacks to using decorators, since they can alter the types of decorated objects, as well as add extra calls to functions, since they add a layer of encapsulation.

In the Python language, there are two basic kinds of decorators: function and class decorators. Function decorators perform automatic name rebinding a function definition time, which enables managing functions and methods, or later calls to them. Class decorators perform automatic name rebinding at class decoration time, which allows managing class instances and their interfaces, as well as later calls to the decorated classes. The implementation aspects of the rebinding will be presented shortly.

Function and class decorators can be used as call and object interface proxies, respectively. Function decorators add a wrapper object to a function, and can forward the function call to the managed object when called. Class decorators add a wrapper object to a class, and can intercept instance creation calls, attribute accesses and method calls, usually forwarding these actions to the managed class as well. Decorators can also be used to manage function and class objects, e.g. to register a function in an API.

Prerequisites

For this post, it is assumed that you are familiar with: Python 2/3 syntax Basic Python knowledge of classes and functions

To run the examples, you must also have Python 2/3 installed in your system.

Using Decorators

A function decorator is a runtime construct, which is executed after the function definition, i.e. after the def block. It automatically rebinds the function to a metafunction (i.e. a function that receives a function as a parameter). The decorator is coded in the line before the function or method definition, using the ‘@’ character followed by the name of the decorator function (or metafunction).

Let’s start with a simple example:

def decorator(F):
    # do some processing with F
    # ...
    # returns F, which is a callable object (i.e. function)
    return F

# The @decorator will automatically rebind myFunc to:
#     myFunc = decorator(myFunc)
# after the myFunc declaration
@decorator
def myFunc(arg):  
    return arg

# myFunc call is equivalent to decorator(myFunc)('hello, world')
# and should print 'hello, world'
print(myFunc('hello, world'))

In this example, myFunc is decorated with the ‘decorator’ metafunction, and it also shows the usage of the ‘@’ character to declare the use of the decorator with a function. Python takes care of rebinding the definition of myFunc to decorator(myFunc) after the function declaration. However, not only functions but classes can also be decorated, and the language provides standard decorators to support the implementation of static methods and class attributes.

Alternatively, the simple example can be rewritten as class wrapper, instead of a function:

class decorator:
    def __init__(self, decorated_func):
        self.func = decorated_func

    # Providing a call method makes decorator objects callable
    def __call__(self, *args):
        return self.func(*args)

# The @decorator will automatically rebind myFunc to:
#     myFunc = decorator(myFunc)
# after the myFunc declaration
@decorator
def myFunc(arg):
    return arg

# myFunc call is equivalent to decorator(myFunc)('hello, world')
# and should print 'hello, world'
print(myFunc('hello, world'))

It works similarly to the function decorator version for functions, however it does not work with class methods. Class methods require an instance to be called, and when the decorated object is called, the arguments list (*args) does not include the class instance.

In order to also support class methods, a better approach is to use a nested wrapper function:

def decorator(F):
    def wrapper_func(*args):
        return F(*args)  # Execute decorated function and return result
    return wrapper_func # Return wrapper_func, which is a callable

@decorator
def myFunc(arg):
    return arg

# myFunc call is equivalent to decorator(myFunc)('hello, world')
# and should print 'hello, world'
print(myFunc('hello, world'))

class EchoClass:
    @decorator
    def echo_value(self, value):
        return value

e = EchoClass()
# e.echo_value call is equivalent to
#     decorator(echo_value)(e, 'hello, world')
# and should print 'hello, world'
print(e.echo_value('hello, world'))

Python binds the class method to the decorator, when the decorator object (wrapper_func) is called, the class instance is passed in the arguments list (*args), and the class method is properly executed.

Nesting Decorators

Several decorators can be applied to any given callable object, and the syntax is exactly the same as is used for a single decorators. The only difference is that one decorator must declared per line, and their calls are nested from top to bottom.

Our nested function example from the previous section is used again here, and we’ll decorate our function and class with three decorators: 

def decoratorA(F):
    def wrapper_func(*args):
        return F(*args)  # Execute decorated function and return result
    return wrapper_func # Return wrapper_func, which is a callable

def decoratorB(F):
    def wrapper_func(*args):
        return F(*args)  # Execute decorated function and return result
    return wrapper_func # Return wrapper_func, which is a callable

def decoratorC(F):
    def wrapper_func(*args):
        return F(*args)  # Execute decorated function and return result
    return wrapper_func # Return wrapper_func, which is a callable

@decoratorA
@decoratorB
@decoratorC
def myFunc(arg):
    return arg

# myFunc call is equivalent to
#     decoratorA(decoratorB)(decoratorC)(myFunc)('hello, world')
# and should print 'hello, world'
print(myFunc('hello, world'))

class EchoClass:
    @decoratorA
    @decoratorB
    @decoratorC
    def echo_value(self, value):
        return value

e = EchoClass()
# e.echo_value call is equivalent to
#     decoratorA(decoratorB)(decoratorC)(echo_value)(e, 'hello, world')
# and should print 'hello, world'
print(e.echo_value('hello, world'))

In this case, our nested decorated function myFunc

@decoratorA
@decoratorB
@decoratorC
def myFunc(arg):
    return arg

is equivalent to

myFunc = decoratorA(decoratorB(decoratorC(myFunc)))

An interesting use case for nested decorators is, for example, nesting a logging decorator with a function call tracing decorator, such that the log is generated for that function and function call statistics can be collected. We’ll see an example of function call tracing in the following sections.

Let’s see another example of nested decorators:

def decorator1(F):
    return lambda F: F + ', how are you today?'

def decorator2(F):
    return lambda F: 'Hello ' + F

@decorator1
@decorator2
def myFunc(name):
    return name

print(myFunc('John Doe'))

A string with the user’s name is passed to myFunc, and we augment its result using decorator1 and decorator2 to provide a greeting message. This example should print “Hello, John Doe, how are you today?”.

Decorator Arguments

Since decorators are callable objects, they can also take any number of arguments. Adding arguments allows for keeping state inside the wrapper, besides the decorated function itself. The following example, shows a very simple decorator with takes two parameters:

def decorator(before_name, after_name):
    def decorator_impl(F):
        # Return callable with concatenation of before_name, F and after_name
        return lambda: before_name + F() + after_name
    return decorator_impl

before = 'Hello '
after = ', how are you today?'

@decorator(before, after)
def greeting():
    return 'John Doe'

print(greeting()) # Should print 'Hello John Doe, how are you today?'

Our decorator takes two parameters, before_name and after_name, which are strings to be concatenated to whatever string the decorated function returns. In order to return a callable, we wrap the concatenation of strings inside a lambda function. Lambda functions are callable objects which do not require naming, i.e. they are anonymous functions. The concatenated result returned by the lambda function is returned and the greeting message is printed to the user.

A More Concrete Example: Tracing Function Calls

Suppose you want to track the number of visits to a given web page. You setup a HTTP server and starts to listen for requests on your routes. For each registered route, you track how many times the response handler for that call is called. With that information, you can see how many times the route has been visited, through the number of times the handler has been called.

In the following example, two routes are registered in the request handler (ReqHandler), one that responds with the current time and another that always respond with ‘Hello World’. Then, we setup the trace_call decorator in each of the registered response handlers, so that we can track how many times each handler is called. In order to track the number of calls, an attribute (calls) is added to the decorator’s wrapper object.

When the decorator object is created, i.e after the declaration of the response handler, the call counter attribute is added to the decorator object. At each time the decorator object is called, the attribute is updated and the number of calls can be tracked.

import http.server
import socketserver
import datetime

PORT = 8000


def trace_call(F):
    def wrapper(*args, **kwargs):
        wrapper.calls += 1
        print("{0} calls have been made to {1}"
              .format(wrapper.calls, F.__name__))
        return F(*args, **kwargs)
    # Add the calls attribute to count how many times
    # the decorated function F is called.
    wrapper.calls = 0
    return wrapper


class ReqHandler(http.server.BaseHTTPRequestHandler):
    # GET route dispatcher
    # All response handlers for routes are registered here
    def do_GET(self):
        if self.path == '/time':
            do_time(self)
        elif self.path == '/hello':
            do_hello(self)


# Response handler for '/time' route
@trace_call
def do_time(request):
    request.send_response(200)
    request.send_header('Content-type', 'text/html')
    request.end_headers()
    # Write the HTML content
    response = bytearray("Current time: " + str(datetime.datetime.now()),
                         encoding='utf-8')
    request.wfile.write(response)

# Response handler for '/hello' route
@trace_call
def do_hello(request):
    request.send_response(200)
    request.send_header('Content-type', 'text/html')
    request.end_headers()
    response = bytearray('Hello, world', encoding='utf-8')
    request.wfile.write(response)


with socketserver.TCPServer(("", PORT), ReqHandler) as httpd:
    print("serving at port", PORT)
    httpd.serve_forever()

Conclusion

In this post, we have covered the roles and the basic syntax of Python decorators. Decorators are powerful abstractions in the language, and are used to declare static methods and class attributes. A more concrete example which tracks the number of calls to functions was also presented, showing one of the capabilites of decorators. In the next posts, we’ll dive deeper into decorators and their applications in the context of classes.
