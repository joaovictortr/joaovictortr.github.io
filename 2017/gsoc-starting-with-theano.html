Starting at GSoC 2017 with Theano

// under python gsoc

I have been selected for GSoC 2017 and I’m glad to work on a project to bring more GPU acceleration to Theano, a sub-org of the Python Software Foundation.

This summer I’ll add support for some GPU libraries in Theano, which will help applications to run potentially faster in GPU-enabled systems.

Why GPU acceleration matters, anyway?

GPUs are becoming increasingly important to accelerate Deep Learning applications. These accelerators provide massive parallelism through lots of concurrent threads. Parallel processing of images, speech and video, for example, speeds up the tasks of training and inference in neural networks.

What is Theano?

Theano is a Python library to handle mathematical expressions with multi-dimensional arrays (tensors) in an efficient way. Theano has been developed in the Machine Institute for Learning Algorithms (MILA), at the University of Montreal, since 2007. It is commonly utilized to implement neural network models, and provides several features, such as:

  • Efficient symbolic differentiation: the library computes the derivatives of functions with one or many inputs.
  • Speed and stability optimizations: Theano can rearrange your computations, in order to obtain more numerical stability. It also support dynamic C code generation, which provides efficient code to perform your computations.
  • Transparent use of a GPU: compute-intensive tasks can be performed on a GPU transparently, and some operations can lift the CPU tasks automatically to the GPU.

As Marek Rei pointed out in a blog post, Theano isn’t actually a machine learning library, but it provides you with means to build your own machine learning models.

My project for this summer consists of three main tasks:

  • Integrate Baidu Research’s warp-ctc library to provide fast connectionist temporal classification (CTC) loss computations. Work on this feature is almost completed (for both CPU and GPU), and should be merged soon. You can find more details on the pull request.
  • Add further GPU-accelerated linear algebra functions to Theano’s new GPU backend. Details from the original project have changed, so I’ll add a follow up post as soon as possible.
  • Implement Spatial Transformer Networks’ operations from the cuDNN library. These networks are composed of three main components: a localization network, a grid generator and a grid sampler. You can find more details on the original paper.

I will add more posts as the project progresses throughout the summer.
